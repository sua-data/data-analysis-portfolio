# 아키텍처

## 전체 흐름
크롤링 → 전처리 → 분석 → 저장 → API → 프론트

---

## 구성

### 1. 크롤링
- 뉴스 데이터 수집

### 2. 전처리
- 텍스트 정제
- 토큰화 (Nori)

### 3. 분석
- 키워드 추출 (TF-IDF)
- 감성 분석

### 4. 저장
- Elasticsearch: 기사, 분석 결과, 로그
- DB: 배치 요약 로그

### 5. 서비스
- FastAPI API 제공

---

## 설계 의도

### DB vs ES 분리
- DB: 핵심 데이터
- ES: 대량 데이터 + 검색

### 배치 처리
- 대량 데이터 효율 처리
